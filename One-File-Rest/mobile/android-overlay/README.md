# Android overlay

The `mobile/android/` project is generated by Capacitor (`cap add android`)
and intentionally gitignored — it's huge, machine-specific, and
reproducible. But there's a handful of native files we *do* need to commit
because they encode our app behavior (share-target intent-filters, FCM
service, signing config). Those live here.

`mobile/scripts/apply-overlay.mjs` runs as part of `pnpm run android:bootstrap`
and:

1. Copies every file under `files/` into `android/`, preserving paths.
2. Patches `AndroidManifest.xml` to register:
   - `<intent-filter>` for `SEND` (single image / text) and `SEND_MULTIPLE`
     so the app appears in the Android share sheet.
   - A deep-link `<intent-filter>` for the `club.elitetok.admin://` scheme
     so OAuth callbacks return into the app.
   - The `EliteTokFcmService` so push notifications display with our
     styled notification + deep-link payload even when the app is killed.
3. Patches `app/build.gradle` to apply `signing.gradle` and the
   `com.google.gms.google-services` plugin.
4. Patches root `build.gradle` to add the google-services classpath.

The script is idempotent: it uses string markers so re-running on a
`cap sync` never duplicates the patches. If a Capacitor upgrade rewrites
the manifest, just re-run `pnpm run android:bootstrap`.

## Files in this overlay

| File                                                      | Purpose                                |
|-----------------------------------------------------------|----------------------------------------|
| `files/signing.gradle`                                    | Reads keystore + passwords from env vars and wires them into the release build type. |
| `files/app/src/main/java/club/elitetok/admin/EliteTokFcmService.java` | Custom `FirebaseMessagingService` that displays a styled notification with deep-link payload. |
