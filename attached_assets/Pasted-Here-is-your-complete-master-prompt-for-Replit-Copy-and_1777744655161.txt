### PART 3: AUTH — ADMIN AUTO-DETECTION AND MAGIC LINK

In `server/routes/auth.ts`, after successful Discord OAuth and user upsert:

**Admin detection:**
1. Read `process.env.ADMIN_DISCORD_IDS`, split by comma, trim whitespace
2. If logged-in user's `discord_id` is in that array → `UPDATE users SET role = 'admin' WHERE discord_id = $1`
3. Else if role is currently not 'staff' → `UPDATE users SET role = 'client' WHERE discord_id = $1`
4. After session save, redirect based on role:
   - `admin` → `/admin`
   - `staff` → `/admin` (staff also use admin panel but with limited permissions)
   - `client` → `/dashboard`

**Magic link login:**
- `GET /auth/access/:token`
- `SELECT * FROM users WHERE portal_token = $1`
- If found: `req.session.userId = user.id`, `req.session.save()`, redirect to `/dashboard`
- If not found: redirect to `/?error=invalid_token`

**Portal token generation:**
- When upserting a user, if they don't have a `portal_token`, generate one with `crypto.randomUUID()` and save it
