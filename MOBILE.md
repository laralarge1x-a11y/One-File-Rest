# Elite Tok Admin — Mobile (Android APK)

A native Android app for Elite Tok Club **staff** that wraps the existing
admin portal with Capacitor, adding the things only a real native app can
do: push notifications when the app is closed, biometric unlock, share-to-
attach from any other app, one-tap camera evidence, and a launcher badge
count.

> Clients are **not** the audience for this app. They keep using the
> responsive website / PWA from Task #2.

## Architecture at a glance

```
┌────────────────────────────────────────────────────────────────────┐
│ Phone                                                              │
│ ┌──────────────────────────────────────────────────────────────┐   │
│ │ APK (Capacitor 6, package club.elitetok.admin)               │   │
│ │                                                              │   │
│ │ ┌─ Native plugins ─────────────────────────────────────────┐ │   │
│ │ │ @capacitor/push-notifications  → FCM, deep-link payload  │ │   │
│ │ │ capacitor-native-biometric     → fingerprint / face      │ │   │
│ │ │ @capacitor/camera              → one-tap photo evidence  │ │   │
│ │ │ send-intent                    → share TO the app        │ │   │
│ │ │ @capacitor/app                 → back button + lifecycle │ │   │
│ │ │ @capacitor-community/badge     → launcher unread count   │ │   │
│ │ │ @capacitor/browser             → in-app Discord OAuth    │ │   │
│ │ └──────────────────────────────────────────────────────────┘ │   │
│ │                                                              │   │
│ │ WebView ─────────────────────────────► server.url            │   │
│ └──────────────────────────────────────────┬───────────────────┘   │
└────────────────────────────────────────────┼───────────────────────┘
                                             │ HTTPS + WSS
                                             ▼
                              ┌────────────────────────────┐
                              │ Replit deployment          │
                              │  ├─ Express API            │
                              │  ├─ Socket.io              │
                              │  └─ services/fcm.ts ──────►│ FCM (Google)
                              └────────────────────────────┘
```

The app is a **3-MB shell**. The actual admin UI is the same React bundle
the website serves — pointing `server.url` (in `mobile/capacitor.config.ts`)
at the deployed admin URL means a normal `vite build` + redeploy ships
every UI change to staff phones on next open. Only changes to native
plugins, splash, icons or Android permissions require a new APK.

## Required environment variables

| Secret                    | Where                | Why                                                      |
|---------------------------|----------------------|----------------------------------------------------------|
| `FIREBASE_SERVER_KEY`     | Replit (server)      | Sends FCM pushes to registered admin devices.            |
| `MOBILE_SERVER_URL`       | Build env (laptop)   | Override `server.url` for local emulator dev.            |
| `ANDROID_KEYSTORE_BASE64` | Replit (build)       | Base64-encoded release keystore (signing).               |
| `ANDROID_KEYSTORE_PASSWORD` | Replit             | Keystore password.                                       |
| `ANDROID_KEY_ALIAS`       | Replit               | Key alias inside the keystore.                           |
| `ANDROID_KEY_PASSWORD`    | Replit               | Key password.                                            |

Without `FIREBASE_SERVER_KEY` the server still runs, the APK still works,
push notifications just become silent no-ops (the `services/fcm.ts`
sender short-circuits with a `[fcm] auth` warning if the key is wrong).

## Dev workflow (live reload onto an emulator)

> Android Studio + JDK 17 are required and must be installed on your
> workstation (not on Replit). All `pnpm run build:android*` scripts will
> fail in the Replit container — they are intentional, run them on your
> laptop after `git pull`.

```bash
# 1) keep the Replit `Start application` workflow running (port 5000)
# 2) on your laptop with the emulator running:
cd One-File-Rest/mobile
MOBILE_SERVER_URL=http://10.0.2.2:5000 pnpm install      # one-time
MOBILE_SERVER_URL=http://10.0.2.2:5000 pnpm run sync     # copies www/ + plugins
MOBILE_SERVER_URL=http://10.0.2.2:5000 pnpm run run:android
```

The emulator opens the app, which loads the laptop's dev server, and any
React save shows up after a pull-down refresh.

For a real phone over USB: same commands, replace `10.0.2.2` with your
laptop LAN IP and put the phone on the same Wi-Fi.

## Release workflow

```bash
# 1) one-time keystore setup on the workstation
keytool -genkey -v -keystore mobile/android-keystore.jks \
  -alias elite-tok-admin -keyalg RSA -keysize 2048 -validity 10000
base64 -i mobile/android-keystore.jks | pbcopy   # macOS
# paste into Replit secret ANDROID_KEYSTORE_BASE64

# 2) export the keystore secrets locally before building
export ANDROID_KEYSTORE_BASE64="…"
export ANDROID_KEYSTORE_PASSWORD="…"
export ANDROID_KEY_ALIAS="elite-tok-admin"
export ANDROID_KEY_PASSWORD="…"

# 3) decode the keystore + build
mkdir -p One-File-Rest/mobile/android
echo "$ANDROID_KEYSTORE_BASE64" | base64 -d > One-File-Rest/mobile/android/release.keystore
cd One-File-Rest && pnpm run build:android:release

# Emits:
#   mobile/android/app/build/outputs/apk/release/app-release.apk   (sideload)
#   mobile/android/app/build/outputs/bundle/release/app-release.aab (Play)
```

The build script does the full bootstrap on every run:
- Installs `mobile/node_modules` if missing (`npm install` inside `mobile/`)
- Generates `mobile/android/` if missing (`cap add android`)
- Applies our overlay (`mobile/scripts/apply-overlay.mjs`) — share-target
  intent-filters, deep-link scheme, FCM service registration, signing
  config, google-services plugin
- Warns if `mobile/android/app/google-services.json` is missing
- For `:release`, also decodes `ANDROID_KEYSTORE_BASE64` →
  `mobile/android/release.keystore` (via `mobile/scripts/decode-keystore.mjs`)
- Runs `cap sync android` then gradle

### Inline upload limits
The `/api/evidence` endpoint accepts base64 data URLs from the native
camera and share intent, but caps individual uploads at **8 MB**. Larger
files should go through the existing Cloudinary upload path on the web
side. This protects the database and prevents an authenticated DoS
through unbounded mobile uploads.

**Sideload to staff phones**: AirDrop / Telegram / Drive the `.apk`,
have the staffer enable "Install unknown apps" for their Files app, tap.

**Play Store internal-testing track**: Play Console → Testing → Internal
testing → Create new release → upload `app-release.aab` → add staff Google
accounts as testers. They install via a Play Store link with no public
listing.

## How push notifications flow

1. Staff opens the APK, biometric unlock fires.
2. `useNativeBridge` calls `initPushNotifications()` which registers with
   FCM and `POST /api/devices/register` with the token.
3. Server stores the token in `device_tokens (user_discord_id, token, …)`.
4. Anything that calls `createNotification()` server-side (a new client
   message, deadline alert, case assignment, etc) fans out to:
   - `notification:new` socket event (in-app toast),
   - `services/push.ts` (web push — PWA / browser),
   - `services/fcm.ts` (FCM — native APK).
5. The APK's notification handler reads `data.url` from the FCM payload
   and deep-links into `/admin/cases/:id` via `useNativeBridge`.

## Bumping the version

Edit `mobile/android/app/build.gradle`:

```gradle
defaultConfig {
    versionCode 2          // monotonically increasing integer
    versionName "1.0.1"    // user-visible
}
```

Then `pnpm run build:android:release`. Play Console rejects uploads with a
`versionCode` ≤ the previous release — bump it every time.

## Secret rotation

| Secret                 | How to rotate                                                                    |
|------------------------|----------------------------------------------------------------------------------|
| `FIREBASE_SERVER_KEY`  | Firebase → Project settings → Cloud Messaging → regenerate, then update Replit secret. Devices keep working — only the server side changes. |
| Release keystore       | **Avoid.** Play Store requires the same signing key for every update. If the key leaks, enroll in *Play App Signing* and request a key reset (manual Google review). |

## Troubleshooting

| Symptom                                  | Likely cause / fix                                                                  |
|------------------------------------------|-------------------------------------------------------------------------------------|
| White screen on launch                   | `server.url` unreachable. Check `mobile/capacitor.config.ts` and your deployment.   |
| No push notifications                    | `FIREBASE_SERVER_KEY` missing on the server, or wrong appId in Firebase console.    |
| Biometric prompt never appears           | `capacitor-native-biometric` not installed in `mobile/`. Re-run `pnpm install && pnpm run sync`. |
| Share intent does nothing                | `send-intent` plugin not installed, or AndroidManifest.xml missing the `<intent-filter>`. Re-run `pnpm run sync` after install. |
| Hardware back closes the app immediately | `attachBackButton()` errored. Check `adb logcat` for `[native]` warnings.           |
| Build fails on Replit                    | Expected — Android SDK isn't available in the container. Build on a workstation.    |
